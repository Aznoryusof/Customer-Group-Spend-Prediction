1. Introduction¶

1.1. Objective¶

The object of this assignment is to the predict total spend and customer groups from a set of survey data using the most appropriate tools.

After building the appropriate model, a machine learning pipeline will be built to generate these new values based on unseen data.

1.2. Adopted approach¶

The testdata.csv will be considered as the holdout set for this task. The reason for the holdout set is to simulate the real-life situation where new data is received, and a prediction would have to be made on the new dataset. Based on the analysis of this assignment, predictions would be made on the holdout set, and outputs on the model performance and final predictions would be available in the model output folder.

The spenddata.csv dataset would be used to find the best models for prediction.

Before carrying out the modelling work, a quick exploration of the training dataset would be done to assess the cleanliness of the dataset, and the target variables for predictions.

To understand the dataset better, it would be ideal to understand in more detail the data collection methods, the data-types available in the dataset, the business use-case for each question collected, and the business environment during data collection.

Despite the above limitations, initial modelling will be conducted, with predictions for total spend being carried out using regression methods and customer groups with classification methods where ever possible.

1.3. Answers to data science questions¶

The following are the answers for questions 1:¶

a. Please explain the choice of metric used
 The metric used for the regression task is the Root Mean Squared Error (RMSE). As the name suggests, RMSE measures the average squared error of the model, and has the same units as the predicted target. The smaller the RMSE, the better the model. The RMSE is more sensitive to errors made by the model, because of the squared term, as compared to the Mean Absolute Error (MAE). Hence, can be considered as a stricter measure of a model's performance.

b. What are the assumptions you made when building this model?
 The main assumption was that discarding the large number of missing columns would not introduce bias into the modelling process. If the reasons for the missing variabe somehow covaries with the missing variable itself, we could introduce some bias if we drop it. This has to be cross-validated with the research design to ensure that dropping largely missing columns would not cause any issues.

c. What were the approaches you considered? Please explain the reason for the technique / approach used as well as the pros and cons.
 Given the limited information available on the dataset, an algorithm called Gradient Boosting Machine was used as it has fewer assumptions made about the data as compared to parametric family of models.

The pros of the algorithm is that it has less tendency to overfit, performs better than a single tree/typical random forest and does not take much compute power to train. In addition, it does not require much data pre-processing (such as normalizing, centering, scaling of features) and can handle missing data well. Another advantage of this model is that it is able to generate feature importance, to help explain the model predictions.

On the other hand, the algorithm has a large number of hyper-parameters to tune in order to find the best model. While a base model works well, to develop a highly accurate model it takes time to find the right combination. Also, it may be less interpretable as compared to an generalised linear model, where statistical inference can be carried out to evaluate the predictors.

d. Please explain under what conditions will the model you choose be not appropriate
 If the task requires statistical inference, this model may not be as useful. Also, if the data contains a significant amount of outliers, this may not be suitable as the model would tend to over fit.

e. How confident are you of the models robustness and how would you explain the models performance?
 The model is fairly robust given that it does not make many assumptions. It has been shown that esembling methods tend to perform better than a single model.

f. Why is your model performing well / not well?
 The model seems to be performing well with a low RMSE of 8.3 and an MAE of 1.8 on the test data.

g. Was any feature engineering required? If yes, what were they. If no, why?
 Currently feature engineering is not required to get a good baseline model. However, the performance of the model can be further improved through choosing the right features to build. Good features could be developed through understanding the factors available in the dataset, and some understanding about the tourism domain.

The following are the answers for questions 2:¶

a. Please explain the choice of metric / evaluation criterion used
 The metric chosen is the logloss and the accuracy. The logloss is typically used to evaluate the performance of a multinomial classifier. The log loss looks at how close a model's prediction is to the actual target values.

The accuracy of the model is simply the number of correctly predicted class over the total number of predictions made.

b. What are the assumptions you made when building this model?
 Given that the GBM was successful in the regression task, it was again used for the classification task. Hence, the assumptions made in the previous tase is the same.

c. What were the approaches you considered? Please explain the reason for the technique / approach used as well as the pros and cons.
 The GBM was used as it has shown good performance in the regression task. The pros and cons are similar to that stated above.

As the classes for the training set is imbalanced, over-sampling strategy was employed by the algorithm for the lower frequency classes.

d. Please explain under what conditions will the model you choose be not appropriate
 Similar to the above.

e. How confident are you of the models robustness and how would you explain the models performance?
 The final model has a 96% overall accuracy, and has low error rate (<1%) for classes with comparatively large sample sizes. It does not perform so well for classes with a low number of sample (error rates of ~1 to 5%).

f. Why is your model performing well / not well?
 One main reason is that the sample size for the lower frequency classes is very low. Hence, errors for those are much higher than the overall rate.

However, while error rates (or recall) may be lower for the lower frequency classes, the model does better on precision for these classes. Hence, in that sense, one can be confident that there would be fewer cases of false positives for the lower frequency classes in this model.

g. Was any feature engineering required? If yes, what were they. If no, why?
 Similar to above.
